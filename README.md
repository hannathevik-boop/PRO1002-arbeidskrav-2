## Refleksjon

Dette prosjektet viser hvordan god struktur bidrar til felles forståelse og delt kodeeierskap i et team. Tydelig dokumentasjon, som README og kommentarer i koden gjør det enklere for andre å sette seg inn i prosjektets struktur og hensikt. Versjonskontroll med Git gjør endringer sporbare over tid og reduserer risikoen for feil når flere jobber på samme kodebase. En ryddig filstruktur og modulær kode øker lesbarheten og gjør vedlikehold enklere. Samarbeidsformer som kodegjennomgang, parprogrammering og tilbakemeldinger fra medstudenter bidrar til kunnskapsdeling og mindre personavhengighet i prosjektet.


________
This project shows how good structure contributes to common understanding and shared code ownership within a team. Clear documentation, such as README and comments in the code, makes it easier for others to understand the project's structure and purpose. Version control with Git makes changes traceable over time and reduces the risk of errors when multiple people are working on the same code base. A tidy file structure and modular code increase readability and make maintenance easier. Collaboration methods such as code reviews, pair programming, and feedback from fellow students contribute to knowledge sharing and less personal dependency in the project.
